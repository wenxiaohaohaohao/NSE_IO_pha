# MAH 项目研究执行总表（冻结 benchmark 版）

## 0. 项目目标与边界

### 0.1 总目标
将当前项目从“研究工作台”推进为“可复查论文工程”：
- 理论主线可完整推导；
- 关键实证对象可计算并可检查；
- 主稿、代码、输出、编译链可复现。

### 0.2 唯一主定义
- 唯一理论主定义文件：`output/doc/MAH_Policy.pdf`
- 非主定义文件（可参考但不覆盖主定义）：历史 notes、草稿、答辩提纲等。

### 0.3 benchmark 硬约束
- 企业类型固定为 `A/B/C`。
- 状态空间固定为一维 `z` 与政策状态 `M in {0,1}`。
- `MAH` 通过 reduced-form 渠道进入：`tau_M(M)`、`kappa_{sj}(M)`、`lambda_B(z,p_m,M)`。
- 不引入政府最优政策、DSGE 化、显式 implementation 顶层控制、额外核心状态维度。

---

## 1. 执行原则

1. benchmark 一致性优先于理论扩展。
2. 数学正确性优先于叙事丰富性。
3. 可实现性优先于模型花哨度。
4. 每个理论对象都要有可计算/可观测映射。
5. 每周必须有“可验证产出物”，避免纯讨论。

## 1.1 当前冲刺优先级（2026-04）

1. 当前主线任务：理论推导 Phase 1-4（先完成数学闭环，再推进实证管道）。
2. `transition/entry` 最小数据管道保留为并行次线，不作为本轮第一优先级。
3. 所有新增写作与证明以 `output/doc/MAH_Policy.pdf` 为唯一主定义。

## 2. 六周推进计划（可压缩）

### 目标
建立唯一主版本、锁定关键口径、定义交付标准。

### 任务
1. 写出《主版本声明与边界说明》。
2. 建立《符号与对象字典》（VA/VB/VC、PiA/PiB/PiC、xA/xB/m、P(M)、e(M)、N(M)、S^O(M) 等）。
3. 锁定 `lambda_B` 口径（二选一并写入决策日志）：
   - 方案 A：pre-MAH 严格为 0；
   - 方案 B：极小截距近似为 0。

### 完成标准（DoD）

## Week 2：理论 Phase 1-3

### 目标
完成从 primitives 到 Bellman 存在性的数学主干。

### 任务
1. Phase 1：Primitives, State Space, Regularity Conditions。
2. Phase 2：三类企业静态最优化（`x_A`、`x_B`、`m`）与优化后流收益。
3. Phase 3：Bellman 系统、算子定义、压缩映射与值函数存在唯一性。

### 完成标准（DoD）
- 形成可插入主稿与附录的成稿段落。
- 每个结论对应明示假设条件。

---

## Week 3：理论 Phase 4-8

### 目标
完成行业转移、进入、市场出清与稳态均衡定义。

### 任务
1. Phase 4：choice regions 与转移矩阵 `P(M)` 可测性与定义。
2. Phase 5：entry 值函数、自由进入互补条件、`e(M)`。
3. Phase 6：contract manufacturing 与 labor market clearing。
4. Phase 7：稳态分布映射与不变测度条件。
5. Phase 8：stationary competitive equilibrium 定义与存在性证明草图。

### 完成标准（DoD）
- 理论链条闭合（值函数、政策、分布、价格、进入同时定义）。
- 附录有对应的存在性与可测性说明。

---

## Week 4：理论 Phase 9-12

### 目标

### 任务
1. Phase 9：定义 `N(M)` 与 `S^O(M)` 并区分数量与结构对象。
2. Phase 10：定义 `bar Pi_C(M)` 与行业生产者侧 surplus `S(M)`。
3. Phase 11：四组比较静态命题（B 型价值通道、创新结果、重组边际、生产者侧歧义）。
4. Phase 12：离散化、内中外三层迭代、收敛准则、模型到矩的映射。

### 完成标准（DoD）

  - `mah_model_main_theory.tex`

## 4. 风险矩阵（当前版）

1. 高风险：主稿源与编译链缺失。
- 影响：无法做主稿级别可复现修改。
- 动作：建立主稿源目录、主 tex 入口、编译说明。

2. 高风险：transition/entry 仍为模板。
- 影响：识别主张无实证支点。
- 动作：优先打通最小数据管道并产出首版矩阵。

3. 中风险：多文档并行导致定义漂移。
- 影响：答辩时口径不一致。
- 动作：建立“主版本字典 + 文档状态标签”。

4. 中风险：`lambda_B` 写法未最终锁定。
- 影响：比较静态、校准与叙述不一致。
- 动作：Week 1 完成二选一并全局同步。
