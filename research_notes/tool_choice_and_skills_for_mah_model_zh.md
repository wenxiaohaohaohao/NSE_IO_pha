# 做 MAH 这篇模型该用什么工具、需要哪些知识：一份实操说明

## 1. 先说结论

如果你的目标是把当前这篇 MAH 文章的模型真正做出来，**最推荐的工具不是 Dynare，而是 MATLAB**。

更准确地说：

- **经验部分**：继续用 `Stata`
- **理论模型与数值求解部分**：优先用 `MATLAB`
- **Dynare**：这篇文章的第一版里，**不推荐作为主工具**

如果以后你数值能力提高了，也可以转到 `Julia`，但对当前阶段而言，`MATLAB + Stata` 是最稳、最可执行的组合。

## 2. 为什么不推荐 Dynare

Dynare 最擅长的是：

- 代表性家庭 / 代表性企业
- 一阶条件系统
- 理性预期
- 稳态附近线性化或高阶近似
- 标准 DSGE、RBC、新凯恩斯模型

而你现在这篇模型的核心不是这个。

你这篇文章更像：

- 异质企业动态模型
- 企业有不同组织状态：`A/B/C`
- 存在进入、退出、状态转移
- 需要求解 `value function`
- 需要写 `Bellman equation`
- 需要求稳态分布
- 需要市场清算

这类模型的主力工具通常不是 Dynare，而是：

- MATLAB
- Julia
- Python

原因很简单：你不是在解一个“宏观均衡条件 + 线性近似”的模型，而是在解一个“动态规划 + 固定点 + 分布演化”的模型。

## 3. 为什么 MATLAB 最适合你现在

### 3.1 MATLAB 是经济学里最常见的数值工具之一

对于你现在这类模型，MATLAB 很适合做：

- value function iteration
- policy iteration
- root-finding
- 固定点迭代
- 离散状态转移矩阵
- 稳态分布求解

而且大多数经济学动态模型的老代码、课程讲义、参考模板，MATLAB 版本最多。

### 3.2 MATLAB 对“第一次做异质企业模型”的人更友好

你现在最缺的不是编程语言本身，而是：

- 动态规划的思路
- 如何把经济学对象翻译成算法
- 如何把 Bellman 方程变成循环和矩阵

MATLAB 在这方面比 Dynare 更直接。你可以一步一步写：

1. 先离散化状态变量 `z`
2. 给定价格猜测 `p_m, w`
3. 迭代求 `V_A, V_B, V_C`
4. 算出最优 policy
5. 算状态转移矩阵
6. 算稳态分布
7. 更新价格直到市场清算

这条路线非常清楚。

### 3.3 MATLAB 和 Stata 的分工也很自然

- `Stata` 负责数据清洗、回归、事件研究、转移矩阵
- `MATLAB` 负责模型求解、比较静态、数值模拟

这正是很多应用经济学和结构经济学论文的标准工作流。

## 4. 这篇模型到底属于哪一类模型

你这篇模型，严格地说，不是“传统宏观 DSGE”，而更接近下面这几类的交叉：

1. `Hopenhayn` 式异质企业动态模型
2. 动态离散选择 / 状态转移模型
3. 带行业均衡的创新模型
4. reduced-form organizational choice model

所以你需要的核心知识不是：

- 新凯恩斯主义
- 货币政策规则

你真正需要的是：

- 动态规划
- Bellman equation
- value function
- stationary distribution
- market clearing
- comparative statics

## 5. 你需不需要 Value Function 和 Bellman Equation？

答案是：**需要，而且这是核心。**

但你不需要一开始就掌握非常高深的连续时间 HJB。

对你来说，第一步用**离散时间 Bellman equation** 完全足够。

### 5.1 Bellman equation 是什么

Bellman equation 的思想非常简单：

> 企业今天怎么选，不仅取决于今天赚多少钱，还取决于这个选择会如何改变未来的状态和未来的价值。

例如，对 `B` 型企业来说：

- 今天可以保持 `B`
- 可以转成 `A`
- 也可以退出

所以它的价值函数不是简单利润，而是：

$$
V_B(z) = \max \{ \text{继续做 B 的当期收益 + 折现后的未来价值},\ \text{转成 A 的价值},\ 0 \}.
$$

这就是 Bellman equation。

### 5.2 为什么你的模型离不开它

因为你文章里最关键的东西都是动态的：

- 企业今天是否进入
- 今天是否转型
- 今天是否退出
- idea 今天是否实施
- `MAH` 会不会改变未来继续经营的价值

这些都不是静态利润最大化能解决的。

## 6. 你需要懂到什么程度？

你现在不需要一次学完整个计算经济学。

你只需要先掌握下面这五个对象。

### 第一：状态变量 `state`

在你的模型里，最基本的状态变量是：

- 企业类型 `s in {A,B,C}`
- 企业能力 `z`

你只要明白：

> 企业当前处于什么类型、能力多强，会影响今天的利润，也会影响未来能不能转型、创新和存活。

这就够了。

### 第二：价值函数 `value function`

价值函数就是：

> 一个企业处在某个状态时，从今天开始一直最优决策下去，能拿到的总贴现价值。

例如：

$$
V_A(z),\quad V_B(z),\quad V_C(z)
$$

它们分别表示不同类型企业在能力状态 `z` 下的长期价值。

### 第三：Bellman equation

就是把价值函数写成“今天收益 + 明天价值”的递归形式。

例如：

$$
V_A(z)=\max_{j \in \{A,B,C,exit\}} \left\{ \Pi_j(z)-\kappa_{Aj}+\beta E[V_j(z')]\right\}.
$$

这句话的意思就是：

- `A` 型企业比较四种选择
- 每种选择都有当期收益和未来价值
- 选总价值最大的那个

### 第四：状态转移

你这篇模型不是只要一个价值函数就完了。

你还要知道：

- `A -> B` 的概率是多少
- `A -> C` 的概率是多少
- `B -> A` 的概率是多少
- `C -> A` 的概率是多少
- 各种类型下，能力状态 `z` 下一期怎么变

这就是状态转移。

### 第五：稳态分布

即使你知道单个企业怎么决策，还不够。

你还要知道全行业里：

- `A` 有多少
- `B` 有多少
- `C` 有多少
- 每类企业在高能力、低能力状态上怎么分布

这叫稳态分布。

这是“GE”的关键部分，因为价格和创新总量是由整个分布决定的，不是由单个企业决定的。

## 7. 你该不该学 HJB？

现在不该把重点放在连续时间 HJB 上。

我建议你按下面顺序学：

1. 先用**离散时间**写 Bellman equation
2. 先把模型求出来
3. 如果以后真要做更高级的连续时间版本，再学 HJB

对你当前论文来说，离散时间已经完全够用了。

## 8. 一个真正可执行的技术路线

### 第一步：先做最小版本

先不要一上来就做很复杂的多产品模型。

你应该先做一个最小版：

- 状态：`(s,z)`
- 类型：`A/B/C`
- 一个价格：`p_m`
- 一个政策变量：`M`
- 一个 Bellman 方程组
- 一个委托生产市场清算条件

这个版本就已经很强了。

### 第二步：数值求解流程

你在 MATLAB 里真正做的事情，大致是：

1. 设定参数
2. 离散化 `z`
3. 猜测 `p_m`
4. 计算 `Pi_A, Pi_B, Pi_C`
5. 迭代求解 `V_A, V_B, V_C`
6. 由价值函数得到组织转移概率
7. 构造总体转移矩阵
8. 求稳态分布
9. 用分布计算委托生产需求和供给
10. 调整 `p_m` 直到市场清算
11. 比较 `M=0` 与 `M=1`

### 第三步：做比较静态

你最先要比较的是：

- `tau_M` 下降后，`B` 的 value function 是否上升
- `B` 的进入是否上升
- `A/B/C` 转移是否变快
- innovation 是否上升
- `C` 的利润是否单调变化

这就是第一版结果。

## 9. 你目前最需要补的经济学知识，不是“宏观”，而是“动态”

这点非常重要。

你可能会觉得“Bellman、value function、GE”听起来很宏观，但对你真正有用的不是 DSGE 宏观训练，而是下面这些：

### 9.1 动态规划

你需要懂：

- 为什么价值函数可以递归写
- 为什么最优决策要比较 continuation value
- 为什么进入、退出、转型必须动态地看

### 9.2 异质企业动态模型

你需要懂：

- Hopenhayn 为什么要有 entry/exit
- 企业分布为什么重要
- 为什么均衡不是一个企业，而是一整个 distribution

### 9.3 数值方法

你需要懂：

- value function iteration 怎么做
- fixed point 怎么找
- 转移矩阵怎么构造
- 稳态分布怎么求

这三块比“宏观 DSGE”更重要。

## 10. 如果按 MIT 顶级经济学训练的标准，会怎么建议你？

如果用非常严格、像 MIT 那样的标准来判断，这里最重要的不是“工具酷不酷”，而是：

### 第一，模型要和问题完全对齐

你的问题是：

- `MAH` 改变研发与生产边界
- 企业在一体化与专业化之间重新选择
- 这会改变 innovation equilibrium

所以模型必须能刻画：

- 组织形式
- 状态转移
- idea implementation
- 行业均衡

Dynare 对这个问题并不自然。

### 第二，模型复杂度必须服从识别和数据

你现在的数据和故事已经足够支持：

- `A/B/C`
- transition
- implementation margin

但还不足以支撑：

- 完整并购谈判
- 不完全契约

所以要克制。

### 第三，先做“对的模型”，再做“漂亮的模型”

对你来说：

- 一个能跑出来、能和转移矩阵对应、能做政策比较静态的 MATLAB 模型

远比

- 一个写得很炫但根本算不动的 Dynare/连续时间模型

更有价值。

这其实就是高水平经济学研究最基本的训练要求：**问题导向，而不是工具导向。**
