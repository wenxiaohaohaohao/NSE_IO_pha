## 更新规则（必须遵守）

1. 只在本目录更新计划文件。
2. 发生口径变更时，主计划与执行管道文档需同步检查一致性：
- benchmark 边界
- `lambda_B` 口径
- 实证对象映射（transition matrix、entry composition、innovation proxy）
3. 每次更新后，在 `plan_change_log_zh.md` 记录：
- 日期
- 修改文件
- 修改摘要
- 触发原因
- 影响范围（理论/实证/答辩口径）

## 建议执行节奏

- 每周固定更新一次计划状态（周五）。
- 重大决策（口径、识别边界、主定义）即时更新，不等周更。
